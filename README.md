# Locality CSV Adapter

Веб-інструмент для перетворення довільних перекладацьких файлів у формат, який очікує Photoshop UXP-плагін [Locality](../Test-fcxhsf).

## Що робить

Приймає на вхід:
- **Google Sheets URL** (опублікований CSV або `/edit` URL — інструмент сам перетворить)
- **Файл** `.xlsx` / `.xls` / `.docx` / `.pdf` / `.csv` / `.tsv`
- **Вставлений CSV** із буфера обміну

Розпізнає й мапить:
- Назви колонок мов (рос/укр/англ) → ISO 639-1 коди (`Русский → ru`, `Турецкий → tr`, `Кыргызский → ky`...)
- Назви ключів-рядків (рос/укр/англ) → стандартизовані ключі (`Название → title`, `Подзаголовок → subtitle`, `Кнопка → cta`...)

На виході — `locs.csv` у форматі плагіна:
```
key,ru,en,tr,kk,ar,...
title,Майская жара,May Heat,...
subtitle,Горячий май...,Red-hot May...,...
```

Можна редагувати мепинг руками у вкладці "Перевірте мепинг" перед експортом.

## Запуск

Інструмент **повністю клієнтський** — нічого не йде на сервер.

### Локально
1. Відкрити `index.html` у браузері (Chrome / Firefox / Edge / Safari).
2. Все.

### Хостинг
Закинути папку на будь-який static hosting:
- **GitHub Pages** — push у репозиторій з `index.html` у корені, увімкнути Pages у Settings.
- **Vercel** — `vercel deploy` (auto-detect static).
- **Cloudflare Pages** — drag-drop папку.
- **Netlify** — drag-drop папку.

CDN-залежності (Papa Parse, SheetJS, mammoth, pdf.js) тягнуться з jsdelivr/unpkg/cdnjs автоматично.

## Підтримувані формати в деталях

### Google Sheets URL
- Якщо URL `/edit#gid=...` — автоматично перетворюється на `/export?format=csv&gid=...`. Працює якщо Sheet **shared with anyone with link**.
- Якщо Sheet приватний — отримаєте 401/403, з підказкою опублікувати: `File → Share → Publish to web → CSV`.
- Можна одразу вставити публічний `/pub?output=csv` URL.

### Excel (.xlsx / .xls / .xlsm)
- Парситься перший аркуш через [SheetJS](https://sheetjs.com).
- Усі рядки + колонки повертаються як 2D-масив.

### Word (.docx)
- Конвертується у HTML через [mammoth.js](https://github.com/mwilliamson/mammoth.js).
- Шукається перша/найбільша таблиця у документі. Її клітинки і стають даними.
- Якщо таблиць немає — помилка з пропозицією експортувати у xlsx.

### PDF
- Витягуються текстові елементи через [pdf.js](https://mozilla.github.io/pdf.js/).
- Групуються за Y-координатою у рядки, потім сортуються за X у колонки.
- **Точність обмежена** — PDF не зберігає семантику таблиць, плагін відтворює її евристично.
- Якщо PDF — скан (растровий), знадобиться OCR (поки не підтримується).

### CSV / TSV / TXT
- Авто-визначення розділювача (кома vs табуляція) за першим рядком.
- UTF-8.

## Алгоритм мапінгу

1. **Перший рядок = заголовки колонок.**
   - Колонка 0 — `key` (ключі-рядки).
   - Колонки 1+ — мови. Кожна нормалізується (lowercase, trim, видаляються дужки) і шукається у словнику псевдонімів. `Кыргызский → ky`, `Filipino → tl`, `Persian (Farsi) → fa`.
2. **Кожен наступний рядок = переклад одного ключа.**
   - Перший стовпчик — назва (`Название`, `Подзаголовок`, `cta`...) — мапиться на стандартний ключ.
   - Решта — переклад у відповідній мові.
3. **Не визначені автоматично** мови/ключі позначаються `[не визначено]` — користувач вписує руками або тискає `пропустити`.

## Розширення словників

Додати нову мову / ключ — відредагувати об'єкти `LANG_ALIASES` і `KEY_ALIASES` у `adapter.js`. Кожен запис — нормалізована назва (lowercase) → стандартний код.

```js
LANG_ALIASES["хорватский"] = "hr";
KEY_ALIASES["юр. дисклеймер"] = "legal";
```
