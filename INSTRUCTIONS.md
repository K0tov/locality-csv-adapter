# Locality CSV Adapter — інструкція користувача

Веб-інструмент, який перетворює перекладацькі шити (від копірайтерів) у CSV для Photoshop-плагіна **Locality**.

---

## Швидкий старт (за 30 секунд)

1. Відкрити веб-сторінку адаптера у браузері.
2. Завантажити файл або вставити URL Google Sheets / CSV.
3. Натиснути **Розібрати →**.
4. Перевірити автоматичний мепинг мов та ключів (зелені бейджі = ОК).
5. Натиснути **⬇ Завантажити locs.csv**.
6. У Photoshop → плагін Locality → **Choose CSV…** → обрати щойно завантажений файл.

Готово — плагін розпізнає переклади і генерує мовні групи в PSD.

---

## Як запустити сам адаптер

Усе працює клієнтсько (нічого не йде на сервер). Три варіанти:

### А. Локально на одній машині
Просто відкрити `index.html` у Chrome / Edge / Firefox. Усе.

> **Увага:** Для Windows використовуйте URL з трьома слешами: `file:///E:/path/...`. Дві слеші не працюють.

### Б. Через локальний http-сервер (рекомендовано)
Якщо `file://` поводиться дивно (буває з Excel-парсером):

```powershell
cd E:\Photoshop_plugins\phpluggins\locality\csv-adapter
python -m http.server 8765
```

Відкрити `http://localhost:8765/`.

### В. Захостити для всієї команди
Закинути папку на будь-який static-hosting:
- **GitHub Pages** — push у репо з `index.html` у корені.
- **Vercel** / **Cloudflare Pages** / **Netlify** — drag-drop папку.

URL буде доступний всім дизайнерам. Вони можуть не ставити нічого на свою машину.

## Крок 1 — завантаження даних

На сторінці три вкладки джерела:

### 🔗 Google Sheets
Вставте URL.
- **Якщо `/pub?output=csv`** (опубліковане посилання) — найпростіше, працює завжди.
- **Якщо `/edit#gid=...`** — інструмент сам конвертує. Працює якщо Sheet **доступний за посиланням** ("anyone with the link can view").
- **Якщо Sheet приватний** — отримаєте 401/403. Опублікуйте: `Файл → Поділитися → Опублікувати в інтернеті → CSV`.

### 📁 Файл (xlsx / docx / pdf / csv)
Drag-drop або клацніть **Виберіть файл**.

| Формат | Як парситься |
|---|---|
| **CSV / TSV** | Авто-визначення розділювача (кома vs табуляція). UTF-8. |
| **XLSX / XLS** | Береться найбільший аркуш. Якщо аркушів кілька — з'являється **selector аркушів** з розмірами (`Wimbledon 2023 (996×44)`), оберіть потрібний. |
| **DOCX** | Шукається перша/найбільша таблиця у документі. Параграфний текст не парситься — лишень table. |
| **PDF** | Best-effort: текст групується по Y/X у рядки/колонки. Точність обмежена, бо PDF не зберігає таблиці семантично. Для скан-PDF не працює (потрібен OCR). |

### 📋 Вставити CSV
Просто вставте дані з буфера. Працює і CSV (комами), і TSV (табами з Excel).

## Крок 2 — перевірка мепингу

Після **Розібрати →** з'являється секція "2. Перевірте мепинг" з двома таблицями:

### Таблиця "Колонки мов"
Кожен рядок — одна колонка вашого файлу. Видно:

```
ОРИГІНАЛ            →   ISO КОД
Русский (RU)        →   ru        [ok]              [пропустити]
English (EN)        →   en        [ok]              [пропустити]
Turkish (TR)        →   tr        [ok]              [пропустити]
Ukraine (UA)        →   uk        [ok]              [пропустити]
Farsi (FA)          →   fa        [ok] [RTL]        [пропустити]
Singal (SN)         →   si        [ok]              [пропустити]
Аргентинский        →   es        [ok]              [пропустити]
Какойсь_дивний      →   (порожньо) [не визначено]   [пропустити]
```

- **Зелений [ok]** — мова автоматично визначена.
- **Синій [RTL]** — права-в-ліво (арабська, фарсі, урду, курдський сорані, іврит).
- **Жовтий [не визначено]** — впишіть код руками (`en`, `ru`, `tr`...) або клікніть **пропустити**.
- **Сірий [пропустити]** — мова виключена з експорту. Клік на **включити** повертає її.

Можна вписувати свій код у поле — наприклад змінити `pt` на `br` для бразильського португальського.

### Таблиця "Ключі (рядки)"
Те саме для рядків. Кожен рядок вашого файлу = один ключ:

```
ОРИГІНАЛ (РЯДОК)    →   КЛЮЧ ДЛЯ PSD
Название            →   title         [ok]
Подзаголовок        →   subtitle      [ok]
Кнопка              →   cta           [ok]
Бонус               →   bonus         [ok]
Период              →   period        [ok]
2 слайд             →   2_слайд       [не визначено]    ← виправте на title чи slide_2
```

**Важливо:** ім'я ключа має точно збігатися з назвою текстового шару у PSD. Якщо в PSD шар названий `title`, ключ повинен бути `title` (з малої букви).

## Крок 4 — використання у плагіні Locality

1. Відкрити Photoshop, відкрити PSD з підготованою master-групою.
2. У плагіні Locality натиснути **Choose CSV…**.
3. Вказати щойно завантажений `locs.csv`.
4. Список чіпів заповниться мовами з вашого файлу.
5. Виділити майстер-групу в Layers panel.
6. Відмітити мови → **Застосувати**.

## Особливі випадки

### Таблиця має кілька секцій (нестандартна структура)
Деякі шити містять кілька банерів стоплено вертикально, наприклад:

```
s          | RU                     | EN
2 слайд    | Заголовок банера №1    | Title of banner 1
3 слайд    | Підзаголовок №1        | Subtitle of banner 1
Кнопка     | Грати                  | Play
1 слайд    | Заголовок банера №2    | Title of banner 2
Кнопка     | Реєстрація             | Register
```

**Locality плагін очікує одна структура = один CSV.** Тут одна `Кнопка` виходить двічі — конфлікт.

**Що робити:**
1. **Найпростіше:** скопіюйте з xlsx тільки рядки одного банера (4-5 рядків) → вкладка "📋 Вставити CSV" → виправте ключі в мепингу (`2 слайд` → `title`, `Кнопка` → `cta`) → завантажте. Один файл = один PSD.
2. **Систематично:** попросіть копірайтерів робити **окремий аркуш на банер** з простою структурою `key,ru,en,...`. Тоді sheet picker дасть вибрати потрібний.

### Дублікати ISO-кодів (Brazil + Portuguese)
Якщо файл має і `Portuguese (PT)`, і `Brazil (BR)` — обидва мапляться у `pt`. Це призведе до колізії в фінальному CSV (дві колонки з одним заголовком).

**Що робити:** у мепингу вручну змінити одну з них:
- Brazil (BR) → код `br` (тоді в Photoshop утвориться група **BR** окремо від **PT**)
- Argentina (AG) → код `ag` (окремо від **ES**)

Це збереже бразильський/аргентинський варіанти як самостійні локалізації.

### Невідома мова
Якщо у списку є мова якою адаптер не знає (`Ілуктоянська`, `Idioma novo`):
- Впишіть ISO-код руками (`xx`).
- Або скажіть розробнику плагіна — додамо в словник.

### Ім'я групи в PSD ≠ коду мови
За замовчуванням плагін Locality робить групи з uppercase коду (`ru → RU`, `kk → KZ`, `sw → SWA` за вашою конвенцією). Якщо хочете інше — змініть `LANG_INFO` у `src/lib/langs.js` плагіна.

### Перший рядок — заголовок типу "Оновлені коди"
Адаптер автоматично пропускає leading-рядки з єдиним непорожнім значенням якщо решта рядків мають по 3+ заповнених клітинок. Якщо все одно не пропустилось — видаліть title-рядок у Sheets і експортуйте ще раз.

## Частим відповіді

**Q: Чи мої переклади не йдуть кудись на сервер?**
A: Ні. Усе виконується у вашому браузері (JavaScript). Адаптер не надсилає жодного байта на сервер. Можна перевірити вкладкою Network у DevTools — побачите лише запити до CDN бібліотек (papaparse, SheetJS, mammoth, pdf.js).

**Q: Чому Sheet з паролем / приватним доступом не вантажиться?**
A: CORS блокує неавторизовані крос-доменні запити. Опублікуйте Sheet (`Файл → Поділитися → Опублікувати в інтернеті → CSV`) або експортуйте у xlsx і завантажте файл напряму.

**Q: PDF не парситься правильно. Чому?**
A: PDF не зберігає семантику таблиць — це просто текст з координатами. Адаптер групує його евристично, але якщо PDF складний (мультиколонки, пробіли в клітинках) — точність низька. Альтернатива: відкрити PDF, копіювати таблицю у Word/Excel, експортувати як xlsx.

**Q: Excel-файл з кирилицею в назві аркуша не вантажиться.**
A: Кирилиця в назвах підтримується, але якщо параметри чи макроси є — спробуйте зберегти як простий .xlsx (без макросів) у Excel.

**Q: Що таке UTF-8 BOM у завантаженому CSV?**
A: Перші 3 байти `EF BB BF` — мітка для Excel/деяких редакторів, що файл UTF-8. Без них Excel може показати кирилицю як "?????". Плагін Locality читає BOM коректно.

**Q: Як редагувати словник мов?**
A: Відкрити `adapter.js`, шукати `LANG_ALIASES = {`. Додавати рядки `"новеназва": "iso"`. Усі ключі — lowercase після нормалізації.

## Корисні шорткати в адаптері

| Дія | Що робить |
|---|---|
| Кнопка **Приклад** | Підставляє показовий CSV у textarea (для тестів) |
| Поле ISO-коду | Можна вписати **будь-який** код, не лише з нашого словника |
| **пропустити** | Виключає колонку/рядок з фінального CSV без видалення з даних |
| **включити** (якщо пропущено) | Повертає рядок назад у CSV |
| **📋 Копіювати** | Кладе CSV у буфер — зручно вставити прямо у Google Sheets для перевірки |

## Якщо щось зламалось

1. Відкрити DevTools (F12) → вкладка Console → подивитись помилки.
2. Перевірити що бібліотеки завантажились: у Console набрати `Papa, XLSX, mammoth, pdfjsLib` — кожна має повернути об'єкт.
3. Якщо CDN недоступний (немає інтернету) — завантажте бібліотеки локально у `lib/` і поправте посилання у `index.html`.

Скрін помилки + назва файлу → розробнику плагіна.
